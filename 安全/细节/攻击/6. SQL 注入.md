## 6. SQL 注入

Sql 注入攻击，是通过将恶意的 `Sql`查询或添加语句插入到应用的输入参数中，再在后台 `Sql`服务器上解析执行进行的攻击

![](D:/%E6%96%87%E4%BB%B6/typora%E5%9B%BE%E7%89%87/ead52fa0-8d1d-11eb-85f6-6fac77c0c9b3.png)

流程如下所示：

- 找出 SQL 漏洞的注入点
- 判断数据库的类型以及版本
- 猜解用户名和密码
- 利用工具查找 Web 后台管理入口
- 入侵和破坏

预防方式如下：

- 严格检查输入变量的类型和格式
- 过滤和转义特殊字符
- 对访问数据库的 Web 应用程序采用 Web 应用防火墙
